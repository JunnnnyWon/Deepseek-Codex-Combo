# User-Perspective E2E QA Plan

## TL;DR
> Summary:      Run a sandbox-only, user-facing E2E QA pass for DeepSeek-Codex-Combo across the built release CLI, installer, proxy, doctor, Codex profile, plugin/model surfaces, orchestration commands, and rollback. The plan deliberately avoids real user-home mutation and makes live DeepSeek checks conditional on an existing `DEEPSEEK_API_KEY`.
> Deliverables:
> - Evidence directory: `evidence/user-e2e-qa-20260608/`
> - State directory: `.dcc/user-e2e-qa-20260608/`
> - Sandbox install/profile/proxy/doctor/rollback receipts
> - Fail-closed receipts for missing key, native provider mode, busy proxy port, and bad plugin/fixture inputs
> - Conditional Codex CLI and live DeepSeek receipts
> Effort:       Medium
> Risk:         Medium - `codex` and real live DeepSeek availability vary by machine, so those paths must record deterministic skip receipts when unavailable.

## Scope
### Must have
- Exercise the local checkout CLI (`node bin/dcc.mjs`) and rebuilt release CLI (`node dist/bin/dcc.mjs`) after `pnpm build`.
- Validate sandbox-first install, generated `deepseek-proxy` profile, provider config, plugin cache, model catalog, marketplace file, agent file, hook command, MCP/LSP describe command, and uninstall rollback.
- Validate `run-dcc-sandbox.command` noninteractively with `DCC_SANDBOX_SKIP_CODEX=1`; if `codex` is unavailable, record a static contract pass instead of hanging.
- Validate proxy background lifecycle, health, mocked `/v1/responses`, fixture transforms, stream fixture, cache diagnostics redaction, busy-port fail-closed behavior, and stopped status.
- Validate `DEEPSEEK_API_KEY` handling: no-key `doctor --live --strict` must fail closed with `DEEPSEEK_API_KEY required`; real live checks run only if the environment already has a key.
- Validate plugin/model regressions: `models`, plugin validation, bad agent model rejection, missing hook rejection, fixture health, generated Codex model metadata fields when `codex` is available, and orchestration/evidence commands.
- Produce cleanup receipts proving the sandbox proxy is stopped, uninstall removed managed Codex state, temp homes are removed, and evidence does not contain secrets.

### Must NOT have (guardrails, anti-slop, scope boundaries)
- Must not run `dcc install` or `dcc uninstall` against the real user home.
- Must not write to `~/.codex`, real `CODEX_HOME`, or user-level Codex config.
- Must not paste, print, commit, or store a real DeepSeek key in evidence.
- Must not publish to npm, call `npx deepseek-codex-combo install`, upgrade dependencies, rewrite fixtures, or edit product source code.
- Must not treat absent `codex` or absent `DEEPSEEK_API_KEY` as human blockers; record deterministic skip evidence.
- Must not leave background proxy processes running after any task.

## Verification strategy
> Zero human intervention - all verification is agent-executed.
- Test decision: tests-after + Vitest and bash CLI scenarios
- QA policy: every task has agent-executed scenarios
- Evidence: `evidence/task-<N>-<slug>.<ext>` plus `evidence/user-e2e-qa-20260608/task-<N>-<slug>.<txt|json>`

## Execution strategy
### Parallel execution waves
> Target 5-8 tasks per wave. <3 per wave (except final) = under-splitting.
> Extract shared dependencies as Wave-1 tasks to maximize parallelism.

Wave 1 (no dependencies):
- Task 1: Create isolated QA state, evidence root, temp home, and dynamic port
- Task 2: Rebuild and package release CLI, then verify help/no-side-effect behavior

Wave 2 (after Wave 1):
- Task 3: Sandbox install/profile/plugin happy path
- Task 4: `run-dcc-sandbox.command` noninteractive launcher coverage
- Task 7: Plugin/model/orchestration regression commands that do not require installed sandbox state

Wave 3 (after Wave 2):
- Task 5: Offline proxy lifecycle, fixtures, cache diagnostics, and busy-port fail-closed checks
- Task 6: Doctor live/key gating and redaction checks

Wave 4 (after Wave 3):
- Task 8: Rollback, cleanup receipts, and no-real-home pollution audit

Critical path: Task 1 -> Task 2 -> Task 3 -> Task 5 -> Task 8
