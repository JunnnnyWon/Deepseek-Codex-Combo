# Supply Chain

Supply-chain controls:

- Keep install-time scripts minimal.
- Show every binary and managed file the installer plans to touch.
- Generate SHA-256 checksums for release files.
- Keep the GitHub Actions workflow running lint, typecheck, unit, integration, and e2e tests.
- Prefer signed release artifacts when publishing outside a local checkout.

The package manifest is generated by:

```bash
node bin/dcc.mjs package --dry-run --out .dcc/release
```

Before publishing a non-dry-run package, verify that every checksum in `.dcc/release/checksums.manifest.json` matches the corresponding file under `.dcc/release/files/`, then run an install/uninstall smoke from `.dcc/release/files` against a temporary `DCC_SANDBOX_HOME`.
